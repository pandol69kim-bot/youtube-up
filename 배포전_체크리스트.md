# 배포 전 체크리스트

## 현재 완료된 MVP

- Next.js + Tailwind 프론트엔드 워크플로 화면
- FastAPI 백엔드 API
- 회원가입/로그인 JWT 발급 API
- 플레이리스트 생성/수정/삭제/목록 API
- 다중 음원 업로드 API
- 트랙 메타데이터/순서 편집 API
- FFmpeg 기반 영상 렌더링 호출부와 챕터 자동 생성
- Pillow 기반 1280x720 썸네일 생성
- YouTube 업로드/예약 발행 mock 어댑터
- 콘텐츠/관리자 통계 API
- Docker Compose 개발 구성
- 프론트엔드 프로덕션 빌드 검증
- 백엔드 API 테스트 검증

## 배포 전 필수 외부 작업

- 운영 서버 또는 Docker 이미지에 FFmpeg 설치
- `.env.example`을 기준으로 운영 `.env` 작성
- `JWT_SECRET`을 강한 난수로 변경
- YouTube Data API v3 OAuth 클라이언트 생성
- `YOUTUBE_UPLOAD_MODE=real` 전환 전 실제 YouTube 업로드 어댑터 연결
- PostgreSQL/RDS 연결 문자열로 `DATABASE_URL` 교체
- S3 버킷/IAM 권한 생성 후 파일 저장소 어댑터 연결
- HTTPS 인증서와 도메인 연결
- 운영 DB 마이그레이션 도구 도입

## 현재 제한

- 이 저장소의 로컬 Windows 환경에는 FFmpeg가 없어 영상 렌더링 API가 명확한 실패 메시지를 반환합니다.
- YouTube 업로드는 기본값이 mock입니다. 실제 업로드에는 OAuth 동의 화면과 refresh token 발급이 필요합니다.
- Redis/Celery, S3, PostgreSQL 운영 연동은 Docker/환경변수 자리에 맞춰 확장 가능한 상태이며, 현재 MVP는 SQLite와 로컬 파일 저장소로 동작합니다.
