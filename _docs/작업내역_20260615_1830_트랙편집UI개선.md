트랙 편집 목록 UI 세 가지 개선: 순서 번호 1부터 표시, 곡 삭제 버튼 추가, 플레이리스트 제목 인라인 편집.

- `frontend/app/api.ts`: `deleteTrack()`, `updatePlaylist()` 함수 추가
- `frontend/app/page.tsx`: UI 변경 및 관련 핸들러 추가

## 변경 내용 상세

### 순서 번호 1부터 표시
- 기존: sort_order 값(0-based)을 직접 editable input으로 표시
- 변경: `index + 1` 정적 텍스트로 표시 (1, 2, 3...)
- 그리드 첫 컬럼 56px → 40px, 헤더 "순서" → "#"

### 곡 삭제 버튼
- Trash2 아이콘 버튼 추가 (각 트랙 행 맨 오른쪽)
- 클릭 시 `DELETE /tracks/:id` 호출 → 파일도 서버에서 함께 삭제
- 삭제 후 tracks 상태와 lyricsOpen Set에서 즉시 제거
- 그리드 컬럼 6개 → 7개: `grid-cols-[40px_1.2fr_1fr_1fr_72px_36px_36px]`
- hover 시 빨간색으로 강조

### 플레이리스트 제목 편집
- "트랙 편집" 패널 헤더에 제목 input 추가
- 플레이리스트 선택 시 해당 제목이 input에 자동으로 채워짐 (useEffect)
- onBlur 또는 Enter로 `PUT /playlists/:id` 저장
- 저장 성공 시 사이드바 플레이리스트 목록도 즉시 갱신

## 주요 결정 사항

- sort_order 편집 기능 제거: 드래그 앤 드롭 없이 숫자만 바꾸는 재정렬은 UX가 불명확하여 정적 표시로 변경
- 플레이리스트 제목 편집 위치: 별도 섹션 대신 트랙 편집 패널 상단에 통합하여 "어떤 플레이리스트를 편집 중인지" 맥락을 함께 제공
