## 작업 내용

플레이리스트 정보를 Markdown 파일로 다운로드하는 기능을 추가했다.

## 변경된 파일

- `backend/app/routers/playlists.py`: `GET /playlists/{id}/export.md` 엔드포인트 추가
- `frontend/app/page.tsx`: 트랙 편집 패널 헤더에 FileDown 아이콘 다운로드 버튼 추가

## Markdown 파일 구성

```
# 플레이리스트 제목

> 설명

**총 n곡 · mm:ss**

---

## 수록곡
| # | 제목 | 아티스트 | 앨범 | 길이 |

---

## 챕터
(코드블록 내 타임스탬프 목록)

---

## 가사
(가사가 있는 트랙만 ### 헤더로 표시)
```

## 주요 결정 사항

- 파일명: `{플레이리스트 제목}.md` (한글 포함, `Content-Disposition: filename*=UTF-8''...` RFC 5987)
- 백엔드에서 생성: 가사·챕터 데이터가 서버에 있으므로 클라이언트 생성보다 일관성 보장
- 다운로드 버튼: `<a href="..." download>` 방식으로 브라우저 기본 다운로드 동작 사용
- 가사 없는 트랙은 가사 섹션에서 자동 제외
