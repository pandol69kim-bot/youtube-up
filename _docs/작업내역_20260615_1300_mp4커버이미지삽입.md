## 작업 내용

MP4 렌더링 시 커버 이미지를 배경으로 사용하는 기능 추가.
커버가 없을 때는 기존 단색 배경 유지.

## 변경 파일

- `backend/app/config.py`: `covers_dir` 설정 추가, 시작 시 디렉토리 자동 생성
- `backend/app/main.py`: `/files/covers` static 마운트 추가
- `backend/app/schemas.py`: `RenderRequest`에 `cover_image_path: Optional[str] = None` 추가
- `backend/app/routers/video.py`: `POST /video/cover-upload` 엔드포인트 추가 (JPG/PNG/WEBP)
- `backend/app/services/media.py`: `render_playlist_video`에 커버 이미지 분기 추가
- `frontend/app/api.ts`: `coverUpload()`, `renderVideo()` 시그니처 업데이트
- `frontend/app/page.tsx`: 커버 이미지 state, 파일 선택 UI, 미리보기, 렌더 전 업로드 로직

## 렌더링 ffmpeg 분기

**커버 이미지 있을 때:**
```
ffmpeg -loop 1 -i cover.jpg
       -f concat -safe 0 -i concat.txt
       -map 0:v -map 1:a
       -c:v libx264 -pix_fmt yuv420p
       -vf "scale=1920:1080:force_original_aspect_ratio=decrease,pad=1920:1080:..."
       -c:a aac -b:a 192k -shortest
       output.mp4
```

**커버 이미지 없을 때 (기존):**
```
ffmpeg -f concat -safe 0 -i concat.txt
       -f lavfi -i "color=c=0x{color}:s=1920x1080:r=30"
       -shortest -c:a aac -b:a 192k -c:v libx264 -pix_fmt yuv420p
       output.mp4
```

## 프론트 UI 플로우

1. "커버 이미지" 섹션에서 JPG/PNG/WEBP 파일 선택 → 미리보기 표시
2. 렌더링 클릭 시 커버 파일이 있으면 먼저 `/video/cover-upload`로 업로드 → 경로 획득
3. 렌더링 요청에 `cover_image_path` 포함 → 이미지 배경 MP4 생성
4. 커버 없으면 기존 단색 배경 렌더링
