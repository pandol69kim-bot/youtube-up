## 작업 내용

브라우저 창이 좁을 때 3열 레이아웃이 단일 컬럼으로 무너지고 패널 구분이 안 되는 시각적 깨짐 문제를 수정했다.

## 변경된 파일

- `frontend/app/page.tsx`: 레이아웃 구조 및 패널 스타일 수정
- `frontend/app/globals.css`: body 배경색 조정

## 원인 및 수정 내용

### 1. 3열 그리드 무너짐
- **원인**: `lg:grid-cols-[260px_1fr_360px]` — `lg` 브레이크포인트(1024px) 미만에서 `grid-cols-1`(단일 컬럼)으로 떨어짐. 스크린샷 기준 창 너비가 ~735px로 조건 미충족
- **수정**: 반응형 조건 제거, 항상 3열 고정
  ```
  grid-cols-[260px_1fr_360px] min-w-[1000px]
  ```
- **좁은 창 처리**: `<main>`에 `overflow-x-auto` 추가 → 1000px 미만 창에서 가로 스크롤

### 2. 패널/헤더 구분 안 됨
- **원인**: 패널 배경 `bg-[#fbfaf5]`와 페이지 배경 `#efeee7`의 명도 차이가 너무 작아 경계가 보이지 않음
- **수정**:
  - 패널: `bg-[#fbfaf5]` → `bg-white`, `border-line` → `border-[#c8c8be]`, `shadow-soft` → `shadow-sm`
  - 헤더: `bg-[#fbfaf5]` → `bg-white`, `border-line` → `border-[#c8c8be]`, `shadow-sm` 추가
  - body 배경: `#efeee7` → `#e8e7e0` (조금 더 진하게)

### 3. CSS 캐시 누적
- **원인**: Next.js dev 서버가 핫리로드를 반복하면서 CSS가 제대로 갱신되지 않는 경우 발생
- **수정**: dev 서버 완전 재시작으로 CSS 재컴파일

## 검증

- 재시작 후 `http://127.0.0.1:3000/` 200 응답 확인
- `min-w-[1000px]`, `grid-cols-[260px`, `shadow-sm`, `bg-white` 클래스 HTML에 정상 포함 확인
