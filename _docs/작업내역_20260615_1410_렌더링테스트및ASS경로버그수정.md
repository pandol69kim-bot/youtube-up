## 작업 내용

렌더링 실제 동작 테스트 수행 중 Windows FFmpeg ASS 자막 경로 파싱 버그를 발견하고 수정했다.

### 버그 원인
`_ffmpeg_ass_path()` 함수가 Windows 드라이브 문자의 `:` 를 `\:` (단일 백슬래시)로 이스케이프했으나, 이 FFmpeg 빌드에서는 `\:` 가 필터 옵션 구분자로 처리되어 `C` 를 filename, `/Users/...` 를 `original_size` 로 잘못 파싱했다.

또한 `ass=path` (위치 인수) 방식 대신 `ass=filename=path` (명시적 키) 방식을 사용해야 했다.

### 수정 내용
- `\:` → `\\:` (이중 백슬래시) 이스케이프로 변경
- `ass={path}` → `ass=filename={path}` 으로 변경

## 변경된 파일

- `backend/app/services/media.py`: `_ffmpeg_ass_path()` 이스케이프 수정 (`\:` → `\\:`), `ass=` → `ass=filename=`

## 주요 결정 사항

FFmpeg Windows 빌드에서 필터 옵션 파싱 시 `\:` (단일 백슬래시+콜론)은 이스케이프로 인식되지 않는다. 반드시 `\\:` (이중 백슬래시+콜론)와 `filename=` 명시적 키를 함께 사용해야 한다.
