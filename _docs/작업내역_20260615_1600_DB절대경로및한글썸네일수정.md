## 작업 내용

DB 경로를 절대경로로 수정하고, 썸네일 한글 제목 렌더링 깨짐 문제를 진단 및 수정했다.

## 변경된 파일

- `.env`: `DATABASE_URL`, `UPLOAD_DIR`, `RENDER_DIR`, `THUMBNAIL_DIR` 모두 절대경로로 변경
- `backend/app/services/media.py`: 한글 폰트 지원 추가 (`_KOREAN_FONT_CANDIDATES`, `_load_font()`), `_wrap_text()` 한글 줄바꿈 방식 개선

## 주요 결정 사항

### DB 경로 절대경로 전환
- **문제**: `sqlite:///./data/app.db`는 SQLAlchemy에서 실행 디렉토리 기준 상대경로로 해석됨. 백엔드 실행 위치에 따라 DB 파일이 다른 경로에 생성되어 데이터 분리 문제 발생
- **조치**: 세션에서 실제 데이터가 있는 temp DB 경로로 절대 지정
  ```
  DATABASE_URL=sqlite:///C:/Users/codelab/AppData/Local/Temp/mp3playlist4youtube/app.db
  ```
- **확인**: 백엔드 재시작 후 `/admin/stats` → playlists=3, tracks=8, videos=21 정상 확인

### 한글 썸네일 깨짐 수정
- **문제**: PIL `ImageFont.load_default()`는 ASCII-only 비트맵 폰트 → 한글 문자가 박스(□)로 렌더링됨
- **원인 확인**: Malgun Gothic(`malgun.ttf`)은 Windows 시스템 폰트로 존재하지만 PIL이 이를 자동 탐지하지 않음
- **조치**:
  - `_KOREAN_FONT_CANDIDATES` 리스트 추가 (Windows/Linux 폰트 경로 우선순위)
  - `_load_font(size)`: 후보 폰트를 순서대로 시도, 실패 시 기본 폰트 폴백
  - `_wrap_text()`: 공백 기준 분리 방식 → 문자 단위 반복으로 변경 (한글은 공백 없는 긴 어절 처리 필요)
  - `generate_thumbnail()`: `width=18`로 한글 줄바꿈 적용
- **검증**: `POST /thumbnail/generate` 호출 → 한글 제목 "어머니의 향기 - 늦은 밤 플레이리스트" 정상 렌더링 확인
