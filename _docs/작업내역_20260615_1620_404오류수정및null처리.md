## 작업 내용

브라우저 콘솔에 찍히던 404 오류를 수정하였다.

## 원인

`frontend/app/page.tsx`의 `refreshTracks()`가 플레이리스트 전환 시마다
`GET /video/latest/{playlistId}`를 호출하는데, 완료된 영상이 없을 때
백엔드가 HTTP 404를 반환하여 브라우저 콘솔에 에러가 노출되었다.

## 변경된 파일

- `backend/app/routers/video.py`
  - `GET /video/latest/{playlist_id}`: 영상 없을 때 404 대신 `null` (HTTP 200) 반환
  - `Optional` import 추가

- `frontend/app/api.ts`
  - `latestVideo` 반환 타입: `Video` → `Video | null`

- `frontend/app/page.tsx`
  - `existing` null 분기 명확히 분리 (TypeScript 에러 해소)

## 결과

- `GET /video/latest/1` → HTTP 200, body: `null` (에러 없음)
- 브라우저 콘솔 404 오류 제거
