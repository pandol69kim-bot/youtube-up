## 작업 내용

YouTube 업로드 401 에러 진단, OAuth2 재인증 스크립트 작성, 실제 업로드 검증까지 수행했다.

## 오류 원인 분석

```
HttpError 401 Unauthorized — reason: youtubeSignupRequired
```

실제 원인은 `youtubeSignupRequired`가 아니라 **OAuth2 refresh_token이 YouTube 스코프 없이 발급된 것**이었다.

- 토큰 갱신 자체는 성공 (`Token refresh: SUCCESS`)
- 그러나 `Scopes: None` → YouTube Data API 권한 없음
- `channels().list(mine=True)` 호출 시 `403 insufficientPermissions` 확인

## 변경된 파일

- `scripts/reauth_youtube.py`: YouTube OAuth2 재인증 스크립트 신규 작성
- `.env`: `YOUTUBE_REFRESH_TOKEN` 재인증 후 새 토큰으로 업데이트됨

## 재인증 스크립트 (`scripts/reauth_youtube.py`)

- 스코프: `youtube.upload` + `youtube`
- `redirect_uri=urn:ietf:wg:oauth:2.0:oob` (콘솔 copy-paste 방식)
- `prompt=consent` 강제 → 항상 새 refresh_token 발급
- 인증 코드 교환 후 `.env`의 `YOUTUBE_REFRESH_TOKEN` 자동 업데이트
- 실행: `python scripts/reauth_youtube.py`

## 주요 결정 사항

- 테스트 업로드는 `privacy_status: private`으로 진행하여 채널에 공개되지 않도록 했다
- 가장 작은 파일(9.2MB, video_id=7)을 선택하여 업로드 시간을 최소화했다
