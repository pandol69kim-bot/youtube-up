## 결론 요약

| 구분 | 판정 |
|------|------|
| 순수 크롬 확장프로그램만으로 전체 구현 | ❌ 불가 |

## 기능별 크롬 확장프로그램 가능 여부

### ✅ 가능한 기능

| 기능 | 이유 |
|------|------|
| YouTube OAuth2 로그인 연동 | chrome.identity API로 OAuth 처리 가능 |
| YouTube Data API v3 호출 | fetch/XHR로 REST API 직접 호출 가능 |
| 유튜브 영상 업로드 (업로드 자체) | YouTube Data API v3 resumable upload 지원 |
| 유튜브 플레이리스트 생성 및 영상 등록 | YouTube Data API v3 Playlists/PlaylistItems 지원 |
| 제목 / 설명 / 태그 / 공개범위 설정 | API 파라미터로 처리 가능 |
| 예약 업로드 (publishAt) | API 파라미터로 처리 가능 |
| 파일 선택 UI (MP3 파일 선택) | HTML input[type=file] 가능 |
| 업로드 진행 상태 표시 | Popup UI 또는 인젝션 UI로 표시 가능 |
| 소규모 데이터 저장 | chrome.storage.local / sync 사용 가능 |
| 유튜브 페이지 UI 조작 (DOM 인젝션) | Content Script로 가능 |
| 챕터 타임스탬프 텍스트 자동 입력 | 계산 후 설명란에 텍스트 삽입 가능 |

---

### ❌ 불가능한 기능 (백엔드 서버 필수)

| 기능 | 불가 이유 |
|------|----------|
| **FFmpeg 기반 영상 생성** | 브라우저에서 FFmpeg 실행 불가 (Node.js / Python 환경 필요) |
| **MP3 → MP4 영상 합성** | 대용량 미디어 처리는 브라우저 메모리 한계 초과 |
| **MoviePy 기반 영상 처리** | Python 라이브러리이므로 서버 필수 |
| **배경 이미지/영상 합성** | 서버사이드 미디어 처리 필요 |
| **자막 오버레이 렌더링** | FFmpeg drawtext 필터 필요 |
| **AI 썸네일 생성** | OpenAI / Gemini API 서버사이드 처리 필요 (API Key 노출 위험) |
| **AI 메타데이터 생성** | API Key 보안 문제로 서버사이드 처리 필수 |
| **PostgreSQL DB** | 크롬 확장에서 외부 DB 직접 연결 불가 |
| **AWS S3 파일 저장** | 서버사이드 처리 필요 (보안 자격증명 노출 위험) |
| **Redis + Celery 비동기 큐** | 서버 인프라 필요 |
| **100곡 이상 대용량 처리** | 브라우저 탭 메모리 제한 (보통 수백 MB) 초과 가능성 |
| **4K 영상 생성** | 서버 GPU/CPU 리소스 필요 |

## 기술적 한계 상세

### 1. 브라우저에서 FFmpeg를 쓸 수 없는가?
- `ffmpeg.wasm`(WebAssembly 포팅)은 존재하나:
  - 크롬 확장프로그램 Content Security Policy (CSP)가 WebAssembly 실행을 기본 차단
  - `manifest_version: 3`에서는 더욱 엄격하게 제한
  - 100곡 이상 처리 시 브라우저 탭이 메모리 부족으로 크래시 가능
  - 처리 속도가 서버 대비 10배 이상 느림
- **결론: 프로덕션 수준 사용 불가**

### 2. API Key 보안 문제
- OpenAI, Gemini, AWS 자격증명을 크롬 확장에 넣으면:
  - 사용자가 확장 소스코드에서 키를 추출 가능
  - 크롬 웹스토어 정책 위반 가능성
- **결론: AI 기능은 반드시 백엔드 서버를 통해 프록시 처리 필요**

### 3. 파일 크기 제한
- 크롬 확장 `chrome.storage.local` 최대 용량: 5MB (unlimitedStorage 권한 요청 시 늘릴 수 있으나)
- MP3 파일 자체를 확장 내부에서 처리하기에는 한계
- **결론: 대용량 파일은 서버로 전송 후 처리 필요**

## 권장 아키텍처: 하이브리드 구조

```
[크롬 확장프로그램]
  └─ 파일 선택 UI (MP3 업로드)
  └─ YouTube OAuth2 처리
  └─ 업로드 설정 폼 (제목/설명/태그/예약)
  └─ 업로드 진행 상태 표시
  └─ API 서버에 작업 요청 & 결과 수신
         ↕ REST API
[백엔드 서버 (FastAPI + Python)]
  └─ MP3 파일 수신 및 S3 저장
  └─ FFmpeg 영상 생성 (Celery 비동기)
  └─ AI 썸네일 생성 (OpenAI / Gemini)
  └─ AI 메타데이터 생성
  └─ YouTube Data API 업로드 처리
  └─ PostgreSQL DB 관리
```

## 순수 크롬 확장프로그램으로만 구현할 경우 가능한 MVP 범위

백엔드 없이 크롬 확장만으로 축소 구현 시 가능한 기능:

- 사전 생성된 MP4 파일 → 유튜브 업로드 자동화
- 업로드 메타데이터 자동 입력 (제목, 설명, 태그)
- 챕터 타임스탬프 텍스트 자동 생성 (음원 길이만 입력받아 계산)
- 유튜브 플레이리스트 자동 생성 및 등록
- 예약 업로드 설정

**단, 영상 생성 자체는 외부 도구(편집 앱 등)로 미리 만들어야 함**

## 최종 권고

| 선택지 | 장점 | 단점 |
|--------|------|------|
| 순수 크롬 확장프로그램 | 설치 간편, 서버 비용 없음 | 영상 생성 불가, AI 기능 제한 |
| 크롬 확장 + 백엔드 하이브리드 | 전체 기능 구현 가능 | 서버 운영 비용 발생 |
| 웹 서비스 (기존 기획대로) | 완전한 기능 구현 | 크롬 확장 없음 |
| **웹 서비스 + 크롬 확장 보조** | **최상의 UX, 완전한 기능** | **개발 비용 증가** |

**권장: 웹 서비스(Next.js + FastAPI)를 메인으로 개발하고, 크롬 확장은 유튜브 계정 연동 및 업로드 편의 기능 보조 도구로 추가 개발**
