## 방법 1 — 로컬 직접 실행 (개발 환경 권장)

### 사전 요건

| 항목 | 버전 |
|------|------|
| Python | 3.12 이상 |
| pip | 설치 확인: `python -m pip --version` |
| FFmpeg | 영상 렌더링 시 필요 (`winget install Gyan.FFmpeg`) |

### 패키지 설치

```powershell
cd d:\_크롬익스텐션\mp3playlist4youtube\backend
python -m pip install -r requirements.txt
```

### 서버 실행

```powershell
cd d:\_크롬익스텐션\mp3playlist4youtube\backend
$env:PYTHONPATH = "."
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

> `uvicorn` 명령이 PATH에 없을 경우 `python -m uvicorn` 으로 실행

### 자동 리로드 (코드 수정 시 재시작)

```powershell
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

### 확인

```
http://127.0.0.1:8000/health   → {"status":"ok","env":"development"}
http://127.0.0.1:8000/docs     → Swagger UI
```

## 방법 2 — Docker 단독 실행

### 이미지 빌드 및 실행

```bash
# 이미지 빌드
docker build -t mp3playlist-backend ./backend

# 컨테이너 실행
docker run -p 8000:8000 \
  --env-file .env \
  -v $(pwd)/uploads:/app/uploads \
  -v $(pwd)/renders:/app/renders \
  -v $(pwd)/thumbnails:/app/thumbnails \
  -v $(pwd)/data:/app/data \
  mp3playlist-backend
```

> Docker 이미지 내부에 FFmpeg가 포함되어 있어 별도 설치 불필요

## 방법 3 — Docker Compose (전체 스택)

### 구성 서비스

| 서비스 | 포트 | 설명 |
|--------|------|------|
| backend | 8000 | FastAPI (FFmpeg 포함) |
| frontend | 3000 | Next.js |
| postgres | 5432 | PostgreSQL 16 (현재 미사용, SQLite 사용 중) |
| redis | 6379 | Redis 7 (현재 미사용, Celery 미연동) |

### 실행

```bash
# 전체 기동
docker compose up

# 백그라운드 실행
docker compose up -d

# 백엔드만 빌드 후 재시작
docker compose up --build backend

# 종료
docker compose down
```

### 주의 사항

- `docker-compose.yml`의 backend 서비스가 `DATABASE_URL=sqlite:///./data/app.db`로 설정되어 있어 **PostgreSQL을 사용하지 않음**
- 실제 PostgreSQL 전환 시 `.env`와 `docker-compose.yml` 모두 수정 필요

## 환경변수 (.env)

```env
APP_ENV=development
BACKEND_CORS_ORIGINS=http://127.0.0.1:3000,http://localhost:3000
DATABASE_URL=sqlite:///./data/app.db
UPLOAD_DIR=./uploads
RENDER_DIR=./renders
THUMBNAIL_DIR=./thumbnails
YOUTUBE_UPLOAD_MODE=mock        # mock | real

YOUTUBE_CLIENT_SECRET=
YOUTUBE_REFRESH_TOKEN=
```
