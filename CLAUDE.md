# MP3 Playlist for YouTube - Claude Instructions

## 작업 후 문서화 규칙 (MANDATORY)

모든 작업이 완료된 후 반드시 `_docs` 폴더에 작업 내역 문서를 작성한다.

### 파일명 형식
```
_docs/작업내역_{날짜}_{시간}_{작업내용요약}.md
```

- `{날짜}`: `YYYYMMDD` 형식 (예: `20260615`)
- `{시간}`: `HHMM` 형식 (예: `1430`)
- `{작업내용요약}`: 영문 또는 한글 키워드, 공백 대신 `-` 사용 (예: `팝업UI개선`, `background-script-fix`)

예시: `_docs/작업내역_20260615_1430_팝업UI개선.md`

### 문서 내용 구조
```markdown
# 작업 내역

- **날짜**: YYYY-MM-DD HH:MM
- **작업자**: Claude

## 작업 내용

(수행한 작업을 간결하게 서술)

## 변경된 파일

- `파일경로`: 변경 내용 요약

## 주요 결정 사항

(중요한 설계 결정이나 선택 이유가 있으면 기록)
```

### 적용 시점

- 코드 수정 또는 신규 파일 생성 후
- 버그 수정 완료 후
- 기능 추가 완료 후
- 설정 변경 후
- 리팩토링 완료 후

> 작업 문서 없이 세션을 종료하지 않는다.
