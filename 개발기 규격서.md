---

## 1. 프로젝트 개요

MP3 음원 파일을 업로드하면 자동으로 플레이리스트 영상을 렌더링하고 YouTube에 발행까지 수행하는 웹 자동화 서비스.

### 주요 기능 흐름

```
음원 업로드 → 트랙 편집(제목/아티스트/앨범/가사) → 영상 렌더링(FFmpeg)
→ 썸네일 생성(Pillow) → YouTube 업로드(OAuth2) → 예약 발행
```

## 7. 핵심 서비스 로직

### 7-1. 영상 렌더링 (`services/media.py`)

1. 플레이리스트 트랙을 sort_order 순으로 조회
2. FFmpeg concat 리스트 파일 생성
3. ASS 자막 파일 생성 (제목 4초, 가사 나머지 구간 균등 분배)
4. 커버 이미지 있으면 `-loop 1 -i cover` → 커버를 배경으로 렌더링
5. 커버 없으면 lavfi `color=` 필터로 단색 배경 생성
6. 출력: 1920×1080, H.264, AAC 192kbps

**ASS 자막 포맷**
- Style: Title (Alignment=8, 상단 중앙, Malgun Gothic 72pt)
- Style: Lyrics (Alignment=2, 하단 중앙, Malgun Gothic 52pt)
- Windows FFmpeg ASS 경로 이스케이프: 드라이브 문자 `:` → `\\:`

**썸네일 생성**
- 1280×720 PNG → JPG (quality=92)
- 스타일 3종: midnight / studio / forest
- 한글 폰트: Malgun Gothic (`C:/Windows/Fonts/malgun.ttf`) → 없으면 Pillow 기본 폰트

### 7-2. YouTube 업로드 (`services/youtube.py`, `routers/youtube.py`)

**모드 전환**: `.env`의 `YOUTUBE_UPLOAD_MODE=real|mock`

**real 모드 흐름**
1. `Credentials(token=None, refresh_token=...)` 생성
2. `creds.refresh(Request())` → access_token 자동 갱신
3. `googleapiclient.discovery.build("youtube", "v3")` 클라이언트 생성
4. `videos().insert(part="snippet,status", ...)` resumable 업로드 (4MB 청크)
5. 예약 발행 시 `privacyStatus=private` + `publishAt` ISO8601

**description 자동 조합** (`routers/youtube.py::_build_description`)
```
{사용자 입력 description}

【수록곡】
00:00 트랙1 - 아티스트
03:05 트랙2

【가사】
▶ 트랙1 - 아티스트
가사 내용...
```

## 9. 개발 서버 실행

```bash
# 백엔드 (포트 8000)
cd backend
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000

# 프론트엔드 (포트 3000)
cd frontend
npm run dev
```

### YouTube OAuth2 재인증

```bash
python scripts/reauth_youtube.py
```
- 브라우저에서 인증 URL 접속 → 코드 붙여넣기
- `.env`의 `YOUTUBE_REFRESH_TOKEN` 자동 업데이트
- 필요 스코프: `youtube.upload`, `youtube`

## 10. 알려진 제약사항

| 항목 | 내용 |
|------|------|
| 렌더링 동기 처리 | 렌더링이 동기 HTTP 요청으로 처리됨. 긴 플레이리스트는 타임아웃 발생 가능 |
| 인증 미적용 | API 엔드포인트에 JWT 인증 미적용 상태 (auth 라우터는 구현됨) |
| 트랙 재정렬 | UI에서 sort_order 직접 편집 불가 (드래그&드롭 미구현). 업로드 순서가 재생 순서 |
| 단일 사용자 | 플레이리스트 생성 시 user_id=1 하드코딩 |
| YouTube 채널 | 인증된 Google 계정에 YouTube 채널이 개설되어 있어야 업로드 가능 |
